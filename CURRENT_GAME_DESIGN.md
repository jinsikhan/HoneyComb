# HoneyComb 현재 게임 디자인 (현황 정리)

리디자인을 위한 **현행 구현 기준**(UI/CSS + 게임 로직 JS) 요약 문서입니다.

## 제품/플레이 경험 한 줄
- **헥스(벌집) 보드**에서 **3개 이상 매치로 제거**하며, 제한시간 안에 **다이아(💎) 목표치**를 모아 레벨업하는 캐주얼 퍼즐.

## 화면 구조(레이아웃)
- **컨테이너 카드**: 중앙 정렬 1-panel UI, `width: 400px` 고정 기반 + 모바일에서 `max-width: 100%`.
- **상단 타이틀 영역**
  - 타이틀: `HoneyComb` (Outfit 폰트, 그라데이션 텍스트)
  - 서브 문구: “클릭↔클릭으로 인접 블록 맞바꾸기 · 3개 이상 맞으면 제거”
- **HUD(Stats bar)**: 한 줄(랩 가능)로 4개 지표 노출
  - ⭐ **레벨** `Lv.<value>`
  - 💎 **다이아 진행도** `<current>/<need>`
  - 🏆 **점수**
  - ⏱ **타이머(초)** — 20초 이하 warning, 10초 이하 danger(깜빡임)
- **진행바(Progress)**: 다이아 수집 진행률을 0–100%로 표시
- **메인 플레이 영역**
  - 캔버스 래퍼: 어두운 패널(내부 글로우/인셋), 둥근 모서리
  - 캔버스: `400 x 560` (aspect ratio 약 1:1.4)
- **하단 안내 문구**
  - 조작 힌트 + “연쇄(콤보)” 설명
- **게임오버 오버레이**
  - 검정 반투명 풀스크린 + “다시 하기” 버튼(주황 그라데이션)

## 비주얼 스타일(아트/브랜딩 톤)
- **전체 톤**: 다크 테마, 꿀/주황 계열 포인트(금빛 하이라이트).
- **배경**: 다크 그라데이션 + radial highlight(주황/보라 계열 미세 글로우).
- **타이포**
  - 제목: Outfit (굵고 큰 로고 느낌)
  - 본문: Noto Sans KR
- **HUD 칩 스타일**: 반투명 블랙 배경 + 얇은 보더, 라운드 12px.
- **색상 팔레트(블록)**: 기본 6색 계열(레벨 따라 사용 색 수 증가)
  - 예: `#e74c3c, #f1c40f, #2ecc71, #3498db, #9b59b6, #e67e22`
- **블록 렌더링**: 각 헥스 타일에 radial gradient로 “젤리/유광” 느낌 + 얇은 외곽선.

## 입력/조작 UX
- **기본 조작 2종**
  - **클릭 → 클릭**: 인접한 두 블록을 교환, 교환 결과 3+ 매치면 제거
  - **드래그(같은 색 연결)**: 같은 색을 인접하게 이어서 3+가 되면 즉시 제거
- **터치 지원**: `touchstart/move/end` 처리, `preventDefault`로 스크롤 방지.

## 코어 룰(게임 플레이)
- **매치 조건**: 연결(인접)된 같은 색 그룹 크기 \( \ge 3 \) 이면 제거.
- **제거/리필**: 제거 애니 → 제거된 칸만 리필 애니로 채움.
- **연쇄(콤보)**:
  - 리필 후 보드에 또 매치가 생기면 자동으로 다음 제거가 이어짐.
  - 표기: 2연쇄 “더블”, 3연쇄 “트리플”, 4연쇄+ “익사이팅/AMAZING”
  - 표시 요소: 상단 콤보 문구 + 중앙 배지(`x2!`, `x3!`, `AMAZING!`) + 점수 팝업.
- **특수 블록(아이템)**
  - 💣 **폭탄**: 자신+주변(6방향) 제거, 폭탄 연쇄 전파 가능(주변 폭탄도 포함)
  - 🚀 **미사일**: 가로 한 줄 제거
  - ✚ **십자**: 가로+세로 제거
- **다이아(💎) 블록**
  - 일부 블록이 다이아 속성을 가질 수 있음
  - 다이아 제거 수가 레벨업 진행도의 핵심
- **제한시간**
  - 레벨별 시간 제한(레벨이 오를수록 증가)
  - 시간 초과 시 게임오버 오버레이 표시

## 진행/스케일링(레벨 디자인)
- **보드 크기 증가(레벨 따라 ROWS/COLS 확장)**: 초반 5x5에서 점진 확대
- **색상 수 증가**: 레벨이 오를수록 사용 색 수가 늘어나 매치 난이도 상승
- **다이아 목표치 증가**: 레벨업에 필요한 다이아 수가 레벨에 따라 증가

## 점수/보상(경제)
- **기본 점수**: 제거된 블록 수 * 10
- **콤보 보너스**: 연쇄 단계에 따라 추가 보너스(연쇄가 높을수록 증가)
- **시각 피드백**: 중앙 `+점수` 팝업, 큰 매치(예: 6개 이상) “대박!” 텍스트.

## 피드백/연출(애니메이션)
- **스왑 애니메이션**: 두 셀이 부드럽게 이동(SWAP_ANIM)
- **제거 애니메이션**: 스케일 다운 + 페이드 아웃(REMOVE_ANIM)
- **리필 애니메이션**: 아래에서 올라오며 스케일 인(REFILL_ANIM)
- **버스트 파티클**: 제거 지점에서 색상 파티클 분출
- **플래시**: 매치 순간 짧은 화면 플래시
- **타이머 임박**: 5초 이하 붉은 오버레이 플리커
- **레벨업 연출**: 중앙 텍스트 + 주변 헥스 링이 퍼지는 애니메이션

## 사운드(현재 에셋/트리거)
- **매치(3+)**: `match.mp3` (아이템이 포함되지 않은 일반 매치)
- **콤보(2연쇄+)**: `combo.mp3` (아이템이 포함되지 않은 일반 연쇄)
- **레벨업**: `levelup.mp3`
- **폭탄 아이템**: `bomb.mp3`
- **미사일/십자 아이템**: 현재 구현은 `Missile.mp3` 를 사용
  - 파일명/정책을 `item.mp3`로 통일하고 싶다면 오디오 모듈에서 교체하면 됨.
- **게임오버**: 별도 전용 SFX 없음(현재는 BGM도 비활성/정지 중심)

## 상태/정보 구조(디자인 관점에서 중요 포인트)
- **한 화면 올인원**: 메뉴/설정/튜토리얼이 UI로 분리되어 있지 않고, 텍스트 힌트로만 안내.
- **HUD의 의미가 명확**: 레벨(⭐), 목표(💎), 결과(🏆), 압박(⏱)이 한 줄에 집약.
- **피드백이 강함**: 콤보 배지/파티클/플래시/레벨업 애니로 “터지는 맛” 강조.

## 리디자인 시 체크리스트(현행 대비)
- **정보 우선순위**: 타이머·다이아 목표·콤보 피드백을 어떻게 재배치/단순화할지
- **튜토리얼/온보딩**: 드래그 제거와 스왑 규칙을 UI 튜토리얼로 분리할지
- **접근성**: 색상만으로 구분되는 블록에 패턴/아이콘/색각 대응을 넣을지
- **사운드 정책**: 아이템/콤보/매치가 겹칠 때 우선순위/동시재생/믹싱 룰 정의
